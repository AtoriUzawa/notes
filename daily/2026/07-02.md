## Notes

* Runtime 更关注"如何执行一次 Agent Loop"，Workflow 更关注"如何组织多个执行节点"，两者属于不同层次的抽象。
* LangGraph 的核心思想不是循环（Loop），而是状态驱动（State-Driven）的节点编排。
* Node 应保持职责单一，负责处理 State 并返回状态更新，而不是维护整个执行流程。
* Workflow 应对应一种 Agent Strategy，而不是通过大量条件分支兼容所有模式。
* 优先学习并理解官方框架的设计思想，比直接封装 API 更有助于长期工程能力的提升。
