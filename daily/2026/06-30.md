## Notes

- Tool 的设计对象首先是 LLM，因此接口应尽可能语义清晰，而不是暴露底层协议细节。
- Provider 更关注"如何实现能力"，Tool 更关注"向 Agent 提供什么能力"，两者职责应保持独立。
- Python 标准库已经能够满足 SMTP 邮件发送的大部分需求，没有必要为了简单功能引入额外依赖。
- Pydantic 不仅提供数据校验，也可以通过类型表达数据语义，例如使用 `SecretStr` 表示敏感信息。
- 抽象应该来源于重复，而不是来源于预期，简单实现往往比提前设计更容易演进。
