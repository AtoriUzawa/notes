## Notes

- SMTP 的核心差异不在 server，而在 security mode（SSL / STARTTLS），这是影响连接方式的关键抽象点
- Tool 的设计应稳定返回“语义结果”，而不是暴露底层协议错误或实现细节
- 配置驱动设计（env → settings → runtime behavior）可以显著降低未来扩展成本
- Email 发送属于典型“边界基础设施能力”，适合用 Tool 进行封装但不适合在 Tool 内做复杂逻辑
- 在 Agent 系统中，“能力定义”比“实现方式”更重要，Runtime 应该屏蔽协议复杂性
